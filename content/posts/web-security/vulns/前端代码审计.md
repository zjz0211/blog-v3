# 前端代码审计 —— 看懂网页里的 JS 在干什么

> 你有没有想过——你打开的每个网页，浏览器都下载了一份完整的"说明书"（HTML+CSS+JS）。
> 有些 CTF 出题人就把关键逻辑、甚至 flag 直接写在这份说明书里，
> 只不过用各种花招把它藏起来了。前端代码审计就是"扒开伪装看真相"。

---

## 场景一：代码长这样，是人写的吗？

**场景描述**：打开一个页面，F12 切到 Sources 面板，发现某个 JS 文件长这样：

```javascript
[][(![]+[])[+[]]+(![]+[])[!+[]+!+[]]+(![]+[])[+!+[]]+(!![]+[])[+[]]]
```

或者这样：

```javascript
ﾟωﾟﾉ= /｀ｍ´）ﾉ ~┻━┻   //*´∇｀*/ ['_']; o=(ﾟｰﾟ)  =_=3; ...
```

别慌，这些都是 JS 混淆，本质还是 JavaScript，只是换了一种"写法"。

**原理**：JS 代码是纯文本发送到浏览器的，开发者怕你直接看懂，就用工具把代码转换成"长得不像代码"的形式。但最终浏览器还是要执行它，所以只要能运行，就能还原。

**实战：JS 混淆类型速查表**

| 混淆类型 | 一眼识别的特征 | 难度 | 原理一句话 | 推荐解码方式 |
|---------|--------------|------|-----------|-------------|
|**JSFuck**| 只有 `[]()!+` 六个符号 | 简单 | 用这 6 个字符拼出 `false`、`true`、`undefined`，再转成字符组装代码 | 在线 JSFuck Decoder |
|**AAEncode**| 大量日文颜文字、`ﾟωﾟﾉ` 之类 | 简单 | 把代码编码成看起来像颜文字的字符串 | Console 执行前截获字符串 |
|**JJEncode**| 方框、数字和符号的混合物 | 简单 | 类似 AAEncode 的变种 | 在线 de4js 工具 |
|**压缩混淆**| 代码被压缩成一行，变量名是 `_0x1234` 之类 | 中等 | 缩短变量名、删除空格、把代码变成一行 | F12 Sources 点 `{}` Prettify |
|**商业混淆器**| 代码里有 `_0xabc123` 和大量控制流平坦化 | 较难 | 用复杂的控制流打乱逻辑，但最终还是要拼出可执行代码 | 调试器逐行断点分析 |

**如何剥开混淆的外衣**：

**方法一：狸猫换太子（最安全）**
把代码里所有的 `eval(` 或 `Function(` 替换成 `console.log(`，然后在浏览器 Console 运行。这样代码不会执行，而是把"真正要执行的代码"打印出来给你看。

```javascript
// 原来：eval(someEncodedString)
// 改成：console.log(someEncodedString)
// 然后 Console 里就会打印出解码后的真实代码
```

**方法二：断点截胡（最可控）**
F12 → Sources → 找到代码 → 在 `eval()` 那行下断点 → 刷新 → 代码跑到断点停下 → 看变量面板里显示的值就是即将执行的代码。

**方法三：在线工具（最省事）**
搜索 `de4js`、`JSUnpacker`、`UnPacker`，直接把混淆代码贴进去，一键还原。支持 JSFuck、AAEncode、JJEncode 等多种格式。

**新手避坑**：
-**绝对不要**在带有真实 Cookie/Token 的浏览器中直接运行来历不明的混淆代码。你不知道它除了解码还会干什么额外操作。
- JSFuck 和 AAEncode 执行后不保证直接"返回原始源码"——它只会按设计执行功能。应该在 `eval()` 之前截获待执行的字符串。
- `console.log` 替换 `eval` 的方法对异步代码不一定有效，如果是 setTimeout 或事件绑定的回调，需要额外处理。

## 场景二：看懂 JS 的执行流程

**场景描述**：页面有个登录表单，输入什么都说"密码错误"。你怀疑前端 JS 里做了校验，找到对应的 JS 文件，但里面代码很多，不知道从哪看起。

**原理**：CTF 中前端 JS 就三种用途——验证用户输入、跟后端交互数据、在前端藏 flag。你不需要成为前端工程师，只需要定位关键代码、理解执行流程。

**实战：三步定位关键代码**

**第一步：找到 JS 文件在哪里**

| 位置 | 怎么找 | 特点 |
|------|--------|------|
| Sources 面板文件树 | F12 → Sources → 左侧展开所有文件 | 能看到所有 JS，包括外部链接和内联脚本 |
| HTML 里的 `<script>` | 查看页面源代码（Ctrl+U）搜 `<script` | 内联脚本直接嵌在 HTML 里 |
| 动态加载的 JS | F12 Sources 里搜文件名 | 有些 JS 是运行时才加载的 |

**第二步：搜关键词**

在 Sources 面板的 JS 文件里按 `Ctrl+F`，搜这些关键词：

```
flag          ← flag 可能直接就在代码里
password      ← 验证密码的逻辑
secret        ← 密钥或隐藏信息
admin         ← 管理员相关的判断
fetch(        ← 跟后端发请求
$.ajax(       ← 跟后端发请求（jQuery 写法）
xmlhttp       ← 跟后端发请求（原生写法）
eval(         ← 要执行的代码藏在这里面
atob(         ← base64 解码操作
```

**第三步：用调试器跟一遍流程**

| 操作 | 作用 |
|------|------|
| 行号上点一下 | 下断点，代码跑到这里暂停 |
| `F10` | 单步跳过——不进入函数内部 |
| `F11` | 单步进入——钻进函数内部看 |
| `F8` | 继续执行到下一个断点 |
| 鼠标悬停变量上 | 看当前变量的值 |
| Console 里输变量名 | 手动查看或修改变量的值 |
| 选中代码段，右键"Evaluate in console" | 立刻算出选中代码的结果 |
| 左下角 `{}` 按钮 | Pretty print——把压缩成一行代码展开成可读格式 |

**实战小技巧**：

```javascript
// 如果看到一个 checkPassword 函数
function checkPassword(input) {
    return input === "s3cr3t";
}

// 直接在 Console 里调它，不用填表单
checkPassword("hello");     // false
checkPassword("s3cr3t");    // true
// 这样你就知道密码是 s3cr3t 了
```

```javascript
// 如果看到代码里有 eval(xxx)
// 在 Console 里打印 xxx 就能看到真正要执行的代码
console.log(xxx);   // 比如输出: "flag{js_is_fun}"
```

**新手避坑**：
- 有些反调试代码会用 `debugger;` 语句不断触发断点，让你没法正常调试。解决办法：右键断点行号选 "Never pause here"，或者用 `Disable JavaScript` 先禁用 JS 再开启。
- 混淆代码里的变量名 `_0x1234` 之类的可以右键选"Rename"重命名，方便理解。
- 如果 JS 有 sourcemap（`.map` 文件），浏览器会自动还原原始代码，你甚至看不到混淆版本——直接在 Sources 里就是可读的源码。

## 场景三：自己动手写几行 JS

**场景描述**：遇到编码过的 flag、需要写脚本发请求、或者在 Console 里操作页面——这时候你需要写一点 JS。

**原理**：CTF 里写 JS 只需要三个场景：解码数据、发请求、跑循环。不需要学完整的 JavaScript，掌握下面这几条就够了。

**实战：极简 JS 速查**

```javascript
// ----- 变量声明（就这两种够用了）-----
let name = "guest";          // 可变
const SECRET = "abc123";     // 不可变，常量

// ----- 函数 -----
function verify(token) {
    return token.length === 16;
}
// 箭头函数简写（看到认识就行）
const double = x => x * 2;

// ----- 条件判断 -----
if (role === "admin") {
    console.log("你有管理员权限");
} else {
    console.log("普通用户");
}

// ----- 循环（CTF 最常用）-----
for (let i = 0; i < 10; i++) {
    console.log(i);   // 输出 0 到 9
}

// ----- 数组操作（破解密码常用）-----
let chars = ["a", "b", "c"];
chars.push("d");                // 加元素 → ["a","b","c","d"]
chars.forEach(c => console.log(c));  // 遍历
chars.map(c => c.charCodeAt(0));     // 每个转 ASCII → [97,98,99,100]
chars.filter(c => c > "b");          // 过滤 → ["c","d"]

// ----- 字符串操作 -----
"hello".split("");              // → ["h","e","l","l","o"]
"a,b,c".split(",");             // → ["a","b","c"]
["a","b"].join("-");            // → "a-b"
"hello".charCodeAt(0);          // → 104（字符 → ASCII 码）
String.fromCharCode(104);       // → "h"（ASCII 码 → 字符）
"hello".substring(1, 3);        // → "el"（截取，不包含结束位置）
"hello".includes("ll");         // → true（是否包含子串）
```

**Console 操作速查表**

| 你想做什么 | 怎么写 | 示例 |
|-----------|--------|------|
| 打印变量 | `console.log(变量)` | `console.log(flag)` |
| 打印详细信息 | `console.table(数组或对象)` | `console.table(users)` 以表格形式显示 |
| 算执行时间 | `console.time("t")` + `console.timeEnd("t")` | 测某段代码跑了多久 |
| 清空 Console | `console.clear()` 或右上角  按钮 | Console 太乱时 |
| 复制结果到剪贴板 | 右键结果选 "Copy" 或 `copy(变量)` | `copy("flag{xxx}")` 直接复制 |
| 保存变量为全局 | 右键变量选 "Store as global variable" | 自动存为 `temp1` 变量 |
| 查看对象属性 | `console.dir(对象)` | 展开对象的所有属性方法 |
| 测量性能 | `console.profile("名称")` + `console.profileEnd()` | 分析性能瓶颈 |

**常用的三组编码解码**

```javascript
// 第一组：Base64（最最常见）
btoa("admin:123456");                       // 编码 → "YWRtaW46MTIzNDU2"
atob("YWRtaW46MTIzNDU2");                   // 解码 → "admin:123456"

// 第二组：URL 编码
encodeURIComponent("a=1&b=2");               // → "a%3D1%26b%3D2"
decodeURIComponent("a%3D1%26b%3D2");         // → "a=1&b=2"

// 第三组：字符串 ⇄ 十六进制
function strToHex(s) {
    return s.split("").map(c => c.charCodeAt(0).toString(16).padStart(2, "0")).join("");
}
function hexToStr(h) {
    return h.match(/.{2}/g).map(b => String.fromCharCode(parseInt(b, 16))).join("");
}
strToHex("admin");    // → "61646d696e"
hexToStr("61646d696e"); // → "admin"
```

**在 Console 里发请求**

```javascript
// 最简单的 GET 请求，拿 flag
fetch("/api/flag")
  .then(resp => resp.text())
  .then(data => console.log(data));

// POST 提交登录
fetch("/api/login", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ username: "admin", password: "123" })
})
  .then(resp => resp.json())
  .then(data => console.log(data));

// 带 Cookie 的请求（请求同源 API 默认带 Cookie）
fetch("/admin/data", { credentials: "include" })
  .then(resp => resp.text())
  .then(data => console.log(data));

// 用 async/await 写更清楚（放在 script 里用）
async function getFlag() {
    let resp = await fetch("/flag");
    let text = await resp.text();
    console.log(text);
}
```

**新手避坑**：
- Console 里用 `fetch` 发请求受同源策略限制——不同端口/协议/域名会报 CORS 错误，这是浏览器保护机制，不是代码写错了。
- `btoa()` 只支持 ASCII 字符，如果包含中文会报错，先用 `encodeURIComponent()` 转一下再 base64。
- Console 里的 `let` 和 `const` 是块级作用域，在 `{}` 外面访问不到。如果遇到 undefined，检查是不是作用域问题。

## JS 混淆类型速查表（完整版）

| 类型 | 样貌 | 字符集 | 复杂程度 | 识别方法 | 推荐工具 |
|------|------|--------|---------|---------|---------|
| JSFuck | `[][(![]+[])` | `[]()!+` 仅 6 个 | 低 | 一眼认出，全是括号感叹号 | JSFuck.com 在线解码 |
| AAEncode | `ﾟωﾟﾉ= /｀ｍ´）ﾉ~` | 日文符号 + 括号 | 低 | 看到颜文字样式的就是 | de4js |
| JJEncode | 类似 AAEncode 的变体 | 方框数字符号 | 低 | 跟 AAEncode 很像 | de4js |
| UglifyJS | 一行超长代码 | 普通 JS 语法 | 中 | 变量名 `a,b,c,d`，无空格 | `{}` Prettify |
| Obfuscator.io | `_0x1234` 大量 | 十六进制变量名 | 高 | 变量全是 `_0x` 开头，代码中有 `\x` 转义 | 调试器逐步跟踪 |

## 知识点总结表

| 知识点 | 一句话 | 核心工具/方法 | 新手常犯错误 |
|--------|-------|-------------|-------------|
| JSFuck 识别 | 只有 `[]()!+` 六个符号的代码 | 在线 JSFuck Decoder / de4js | 直接执行来历不明的混淆代码 |
| AAEncode 识别 | 代码里有 `ﾟωﾟﾉ` 等颜文字 | de4js / 替换 eval 为 console.log | 以为执行后返回的就是原始源码 |
| 通用反混淆 | 把 `eval(` 换成 `console.log(` | Sources 面板 + Console | 没在 eval 前截获就运行了 |
| 定位关键代码 | 搜 flag/password/fetch/eval | Ctrl+F 在 Sources 里搜 | 只在页面上看，不切到 Sources 面板 |
| 调试器使用 | F10 单步、F11 进入、F8 继续 | Sources 面板断点 | 忘记下断点就刷新，代码跑完了 |
| base64 编解码 | `btoa()` 编码、`atob()` 解码 | Console 直接调用 | 不知道 btoa 不支持中文 |
| Console 发请求 | `fetch(url).then(r=>r.text()).then(console.log)` | Console 面板 | 跨域请求被 CORS 拦住时以为是代码问题 |
| 字符串转 ASCII | `charCodeAt()` / `fromCharCode()` | Console 调用 | 混淆了字符和 ASCII 码的概念 |
| 关键点 | 所有前端代码都可读、可改、可绕过 | 开发者工具 + Console | 以为混淆过的代码就看不穿了 |
