# 1. 前端代码审计

前端代码审计就是看懂网页的JavaScript在干什么。有些题目会把关键逻辑写在前端，用JSFuck等混淆手段藏起来。

## 1.1 JS 混淆

前端 JS 代码是明文送到浏览器的，开发者为了不让别人直接看懂逻辑，会用各种方式混淆代码。CTF 题目的源码里也经常藏着混淆过的 JS，用来隐藏关键逻辑或 flag。

**核心思路：** 能在浏览器中运行的 JS 混淆代码，运行时总要恢复出足够的值和控制流，因此通常可以通过静态分析或调试逐步还原。不要直接在带有真实 Cookie、Token 的浏览器环境中执行来历不明的混淆代码，优先使用隔离环境、断点或把危险执行点改成输出。

**常见类型和识别方法：**

### 1.1.1 JSFuck

JSFuck 只用 6 个字符就能写出任意 JavaScript 代码：`[` `]` `(` `)` `!` `+`。原理是通过这 6 个字符拼出 `false`、`true`、`undefined` 等基础值，再通过类型转换拼出字母、单词，层层组装成完整代码。

一眼就能认出来——代码全是方括号和感叹号，类似：

```javascript
[][(![]+[])[+[]]+...]
```

**解码方式：**

- 可以在隔离的浏览器环境中执行并观察返回值或副作用；如果代码内部会调用 `eval()` / `Function()`，应先下断点或改成输出，避免直接执行未知逻辑
- 也可以用在线工具（搜索 "JSFuck decoder"），贴进去自动还原

### 1.1.2 AAEncode / JJEncode

AAEncode 把 JS 代码编码成一堆日文颜文字或看起来像乱码的字符。常见特征是代码里大量出现 `ﾟωﾟﾉ`、`oﾟ｀oﾟ` 这样的表情符号。

JJEncode 是类似思路，把代码变成一坨方框、数字和符号的混合物。

**解码方式：**

和 JSFuck 一样，可以在隔离环境中调试；执行编码后的代码只会按其设计运行 payload，不保证直接"返回原始源码"，更稳妥的做法是在 `eval()` / `Function()` 前截获待执行字符串。

### 1.1.3 通用反混淆技巧

除了直接运行，还有几种方法能看到混淆代码的真面目：

- 把代码里的 `eval(` 换成 `console.log(`（不执行，只打印要执行的代码），再在 Console 运行
- 用在线反混淆网站（如 `de4js`、`unpacker`），支持多种混淆格式自动识别
- 用浏览器调试器逐步跟踪：F12 → Sources 面板 → 左下角点 `{}` 按钮（Pretty print，把压缩成一坨的代码展开成可读格式）→ 在 `eval()` 或 `Function()` 语句的行号上点击下断点 → 刷新页面让代码跑到断点处停下 → 鼠标悬停在变量上就能看到当前值，或者选中表达式右键 "Evaluate in console" 看结果

以上方法不仅适用于 JSFuck 和 AAEncode，也适用于大部分前端 JS 混淆。复杂的商业混淆器（如 javascript-obfuscator）也能用调试器理清逻辑。

## 1.2 看懂 JS 代码运行

CTF 题目中的 JS 代码通常就三种用途：验证用户输入、跟后端交换数据、在前端藏 flag。看懂 JS 不需要精通前端，只需要能定位关键代码、理解执行流程。

**去哪看 JS 代码：**

1. F12 → Sources 面板 → 左侧文件树列出页面加载的所有 JS 文件，点开直接读
2. 在代码里 `Ctrl+F` 搜索关键词：`flag`、`password`、`secret`、`admin`、`fetch(`、`$.ajax(`、`xmlhttp`
3. 有些 JS 嵌在 HTML 的 ``<script>`` 标签里，不在单独文件里，在 Elements 面板右键页面 → "View page source" 就能看到

**看懂执行流程：**

CTF 里不需要读懂每一行，关键是多打 `console.log`——在代码里任意位置插入 `console.log(变量名)`，刷新页面就能在 Console 看到当时的值。

常用调试手段：

- **在 Console 手动调函数**：如果代码里有个 `function checkFlag(input)`，直接在 Console 敲 `checkFlag("test")` 就能看它的返回结果，不用填表单
- **断点跟值**：在 Sources 面板某个行号点一下就是断点，代码跑到那里会暂停，鼠标悬停在变量上就能看当前值。继续按 `F10`（单步跳过）或 `F11`（单步进入）逐步往下走
- **搜 `eval`**：如果看到代码里有 `eval(xxx)`，说明 xxx 会被当作代码执行——打印 `xxx` 就能看到真正要执行的逻辑，这是 CTF 里最常见的藏逻辑手段

## 1.3 编写简单的 JS 代码

CTF 里写 JS 基本就两个场景：一是在 Console 里写几行操作页面，二是写个解码/爆破脚本。用到的语法很少，不需要学全套。

**极简语法速查：**

```javascript
// 变量
let a = 1;
const b = "hello";

// 函数
function add(x, y) { return x + y; }

// if 判断
if (a === 1) { console.log("yes"); } else { console.log("no"); }

// for 循环（最常用）
for (let i = 0; i < 10; i++) { console.log(i); }

// 数组
let arr = [1, 2, 3];
arr.push(4);           // 加元素
arr.forEach(v => console.log(v));  // 遍历
arr.map(v => v * 2);   // 每个元素处理后生成新数组

// 字符串操作
"hello".split("");      // → ['h','e','l','l','o']
"a,b,c".split(",");     // → ['a','b','c']
["a","b"].join("");     // → "ab"
"hello".charCodeAt(0);  // → 104（字符转 ASCII）
String.fromCharCode(104); // → "h"（ASCII 转字符）
```

**常用编码解码：**

```javascript
// base64
btoa("admin:123456");   // 编码 → "YWRtaW46MTIzNDU2"
atob("YWRtaW46MTIzNDU2"); // 解码 → "admin:123456"

// URL 编码
encodeURIComponent("a=1&b=2");   // → "a%3D1%26b%3D2"
decodeURIComponent("a%3D1%26b%3D2"); // → "a=1&b=2"

// 字符串 ↔ 十六进制
"hello".split("").map(c => c.charCodeAt(0).toString(16)).join("");
// → "68656c6c6f"
```

**在 Console 里发请求：**

```javascript
// 最简单的 GET
fetch("/api/flag").then(r => r.text()).then(console.log)

// POST 带数据
fetch("/api/login", {
  method: "POST",
  headers: { "Content-Type": "application/x-www-form-urlencoded" },
  body: "username=admin&password=123456"
}).then(r => r.json()).then(console.log)

// 带 Cookie 的请求
fetch("/admin/secret", { credentials: "include" }).then(r => r.text()).then(console.log)
```
