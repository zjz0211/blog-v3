# 1. 密码爆破

密码爆破的核心不是暴力穷举，而是根据题目线索缩小候选范围——伪随机数的种子、哈希特征、已知的密码格式都是突破口。

密码爆破的核心是根据题目给出的条件缩小候选范围，再枚举所有可能值进行验证。

CTF Web 中常见的验证方式有：

1. 根据伪随机数的种子重新生成验证码或 Token。
2. 计算候选明文的哈希，与目标哈希比较。
3. 根据已知格式生成字典，再交给脚本、BurpSuite 或 Hashcat 使用。

## 1.1 伪随机数

### 1.1.1 伪随机数基础

普通伪随机数由固定算法生成。只要随机数算法、种子和调用顺序相同，产生的随机数序列就相同。

例如：

```python
import random

rng = random.Random(123456)

print(rng.randint(0, 999999))
print(rng.randint(0, 999999))
```

每次使用相同种子运行，得到的结果都相同。

CTF Web 中常见的弱种子有：

| 种子来源 | 爆破方式 |
|---|---|
| 秒级时间戳 | 枚举目标时间前后几秒 |
| 毫秒级时间戳 | 枚举目标时间前后几千毫秒 |
| 用户 ID | 直接读取或枚举用户 ID |
| 短数字 | 枚举有限数字范围 |
| 固定种子 | 直接复现随机数序列 |

### 1.1.2 时间种子爆破

假设后端使用当前秒级时间戳作为种子：

```python
import random
import time

seed = int(time.time())
rng = random.Random(seed)

code = f"{rng.randint(0, 999999):06d}"
```

如果已经知道一次验证码和大概生成时间，就可以枚举附近的时间戳。

```python
import random

# 根据请求时间、日志或 HTTP Date 响应头确定
center_time = 1710000000
known_code = "123456"

for seed in range(center_time - 10, center_time + 11):
    rng = random.Random(seed)
    code = f"{rng.randint(0, 999999):06d}"

    if code == known_code:
        print(f"[+] 找到种子：{seed}")

        next_code = f"{rng.randint(0, 999999):06d}"
        print(f"[+] 下一次验证码：{next_code}")
        break
```

这里找到种子后，应继续使用当前的 `rng` 对象生成下一个值。重新创建对象会让随机数序列从头开始。

如果后端使用毫秒级时间戳：

```python
seed = int(time.time() * 1000)
```

可以枚举目标时间前后几千毫秒：

```python
import random

center_ms = 1710000000123
known_code = "123456"

for seed in range(center_ms - 3000, center_ms + 3001):
    rng = random.Random(seed)
    code = f"{rng.randint(0, 999999):06d}"

    if code == known_code:
        print(f"[+] 找到毫秒种子：{seed}")
        break
```

### 1.1.3 经过哈希处理的随机数

有些题目会对随机数计算 MD5，再把结果作为密码重置 Token：

```python
import hashlib
import random
import time

rng = random.Random(int(time.time()))
value = str(rng.randint(0, 999999))
token = hashlib.md5(value.encode()).hexdigest()
```

虽然最终 Token 是完整的 32 位 MD5，但原始随机数只有一百万种，所以仍然可以枚举时间种子。

```python
import hashlib
import random

target = "【目标 MD5】".lower()
center_time = 1710000000

for seed in range(center_time - 10, center_time + 11):
    rng = random.Random(seed)
    value = str(rng.randint(0, 999999))
    digest = hashlib.md5(value.encode()).hexdigest()

    if digest == target:
        print(f"[+] 找到种子：{seed}")
        print(f"[+] 原始随机数：{value}")
        break
```

对弱随机数使用 MD5 或 SHA256，不会增加原始随机数的搜索空间。

### 1.1.4 常见注意点

1. 必须复现相同的随机数算法、种子和调用顺序。
2. Python 的 `random`、PHP 的 `mt_rand()` 和 Java 的 `Random` 不能直接混用。
3. `randint()`、`random()`、`randrange()` 等函数消耗随机状态的方式可能不同。
4. 如果生成 Token 前还调用过其他随机函数，脚本也要执行相同次数的调用。
5. 短验证码可能对应多个种子，最好使用两个连续输出验证。
6. 注意区分秒、毫秒和微秒时间戳。
7. Python 的 `secrets`、PHP 的 `random_int()` 和 `random_bytes()` 属于安全随机数接口，正常情况下不能用时间种子预测。

## 1.2 哈希爆破

### 1.2.1 哈希爆破基础

哈希爆破的流程是：

```text
枚举候选明文 → 计算哈希 → 与目标哈希比较
```

开始爆破前，应先确认：

1. 哈希算法。
2. 明文长度和字符范围。
3. 是否存在盐值。
4. 盐值放在密码前还是密码后。
5. 是否进行了多次哈希。
6. 使用的字符编码。

常见哈希长度：

| 哈希 | 十六进制长度 |
|---|---:|
| MD5 | 32 |
| SHA1 | 40 |
| SHA256 | 64 |

32 个十六进制字符很可能是 MD5，但不能只根据长度就确定算法，最好结合源码判断。

### 1.2.2 完整 MD5 爆破

假设题目给出：

```text
e10adc3949ba59abbe56e057f20f883e
```

并且已知原文是六位数字，可以枚举 `000000` 到 `999999`。

```python
import hashlib

target = "e10adc3949ba59abbe56e057f20f883e"

for number in range(1_000_000):
    password = f"{number:06d}"
    digest = hashlib.md5(password.encode()).hexdigest()

    if digest == target:
        print(f"[+] 找到原文：{password}")
        break
else:
    print("[-] 没有找到")
```

运行结果：

```text
[+] 找到原文：123456
```

这里使用：

```python
f"{number:06d}"
```

是为了保留前导零，防止漏掉 `000123` 这样的密码。

### 1.2.3 不完整 MD5 爆破

假设题目给出的 MD5 是：

```text
e10adc3949ba59abbe56e057f20f88??
```

其中 `?` 表示该位置未知。

不需要先枚举缺失的哈希字符。可以计算每个候选明文的完整 MD5，然后只比较已知位置。

```python
import hashlib

pattern = "e10adc3949ba59abbe56e057f20f88??".lower()

def match_digest(digest, pattern):
    if len(digest) != len(pattern):
        return False

    return all(
        expected == "?" or expected == actual
        for actual, expected in zip(digest, pattern)
    )

for number in range(1_000_000):
    password = f"{number:06d}"
    digest = hashlib.md5(password.encode()).hexdigest()

    if match_digest(digest, pattern):
        print(f"[+] 候选原文：{password}")
        print(f"[+] 完整 MD5：{digest}")
```

如果只知道 MD5 前缀，可以使用：

```python
if digest.startswith("e10adc3949ba"):
    print(password, digest)
```

如果只知道后缀，可以使用：

```python
if digest.endswith("f20f883e"):
    print(password, digest)
```

不完整哈希可能匹配多个候选，因此脚本通常应该输出全部结果，再通过网站或其他条件验证。

### 1.2.4 带盐和多次 MD5

假设后端计算方式是：

```text
md5(password + salt)
```

对应脚本：

```python
import hashlib

target = "【目标 MD5】".lower()
salt = "ctf2026"

for number in range(1_000_000):
    password = f"{number:06d}"
    data = password + salt
    digest = hashlib.md5(data.encode()).hexdigest()

    if digest == target:
        print(f"[+] 找到密码：{password}")
        break
```

如果后端使用：

```text
md5(salt + password)
```

脚本就要改成：

```python
data = salt + password
```

双重 MD5 可以写成：

```python
first = hashlib.md5(password.encode()).hexdigest()
digest = hashlib.md5(first.encode()).hexdigest()
```

常见构造包括：

```text
md5(password)
md5(password + salt)
md5(salt + password)
md5(md5(password))
md5(username + ":" + password)
```

拼接顺序、分隔符或编码不同，最终哈希也会完全不同。

### 1.2.5 常见注意点

1. 先根据源码确认哈希算法和输入格式。
2. 注意目标哈希的大小写，可以统一转换成小写比较。
3. 注意密码中的前导零、空格和换行。
4. 哈希缺失位置越多，出现多个候选的概率越高。
5. 部分哈希匹配时不要找到第一个结果就退出。
6. 已知明文格式时，只枚举未知部分。
7. 搜索空间很大时，应使用 Hashcat，而不是一直使用 Python 单线程。

## 1.3 字典生成和使用

### 1.3.1 根据已知条件生成字典

字典不一定是网上下载的弱密码文件，也可以根据题目条件自己生成。

例如已知密码格式为：

```text
admin + 四位数字
```

可以生成：

```text
admin0000
admin0001
admin0002
...
admin9999
```

如果已知密码为：

```text
12??56
```

并且 `?` 都是数字，只需要生成 100 个候选，不需要枚举整个六位密码空间。

### 1.3.2 使用 Python 生成字典

生成 `admin0000` 到 `admin9999`：

```python
with open("wordlist.txt", "w", encoding="utf-8") as file:
    for number in range(10000):
        password = f"admin{number:04d}"
        file.write(password + "\n")
```

如果每个位置满足不同条件，可以分别设置字符集。

例如：

```text
第 1 位是大写字母
第 2 位是数字
第 3 位只能是 a、b、c
第 4 位只能是 @ 或 _
```

生成脚本：

```python
import itertools
import string

charsets = [
    string.ascii_uppercase,
    string.digits,
    "abc",
    "@_",
]

with open("wordlist.txt", "w", encoding="utf-8") as file:
    for chars in itertools.product(*charsets):
        password = "".join(chars)
        file.write(password + "\n")
```

候选数量等于每个位置字符数量的乘积：

```python
total = 1

for charset in charsets:
    total *= len(charset)

print(f"总候选数量：{total}")
```

生成字典前应先计算候选数量，防止生成过大的文件。

### 1.3.3 使用 Crunch 生成字典

生成所有六位数字：

```bash
crunch 6 6 0123456789 -o 6digits.txt
```

生成所有四位数字：

```bash
crunch 4 4 0123456789 -o 4digits.txt
```

如果字符集包含小写字母和数字：

```bash
crunch 4 6 abcdefghijklmnopqrstuvwxyz0123456789 -o wordlist.txt
```

字符集和长度越大，生成的字典文件越大。使用前应先估算候选数量和磁盘空间。

### 1.3.4 在 Python 中使用字典

使用字典爆破 MD5：

```python
import hashlib

target = "【目标 MD5】".lower()

with open("wordlist.txt", "r", encoding="utf-8", errors="ignore") as file:
    for line in file:
        password = line.rstrip("\r\n")
        digest = hashlib.md5(password.encode()).hexdigest()

        if digest == target:
            print(f"[+] 找到原文：{password}")
            break
    else:
        print("[-] 字典中没有找到")
```

这里只删除行尾的换行符，不建议直接使用：

```python
line.strip()
```

因为 `strip()` 还会删除密码开头和结尾本来存在的空格。

### 1.3.5 在 BurpSuite 和 Hashcat 中使用字典

如果需要把候选密码提交给 Web 接口，可以使用 BurpSuite Intruder：

1. 抓取登录或验证请求。
2. 发送到 Intruder。
3. 标记密码、验证码或 Token 参数。
4. 在 Payloads 中加载字典。
5. 根据状态码、响应长度、跳转位置或关键文字判断结果。

如果已经获得哈希，优先进行离线爆破，不需要向服务器发送大量请求。

Hashcat 使用字典爆破 MD5：

```bash
hashcat -m 0 -a 0 hash.txt wordlist.txt
```

其中：

```text
-m 0    表示 MD5
-a 0    表示字典攻击
```

直接爆破六位数字 MD5：

```bash
hashcat -m 0 -a 3 hash.txt ?d?d?d?d?d?d
```

其中 `?d` 表示一位数字。

### 1.3.6 常见注意点

1. 能够离线爆破时，优先使用 Python 或 Hashcat。
2. BurpSuite 属于在线爆破，速度慢，还可能受到限速和验证码影响。
3. 根据题目条件生成小字典，比直接使用超大字典更高效。
4. 每个未知位置可以使用不同的字符集。
5. 使用 `itertools.product()` 时不要先转换成列表，否则可能占用大量内存。
6. 生成字典前先计算候选数量和大概文件大小。
7. 如果只是离线计算哈希，可以边生成候选边计算，不一定要先写入字典文件。
8. 注意密码中的前导零、空格、换行和字符编码。
