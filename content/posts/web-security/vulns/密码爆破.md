# 密码爆破

> 爆破的核心**不是暴力穷举**，而是根据题目线索缩小候选范围。一条线索可能把搜索空间从百万级降到几十个。

---

## 一、场景

CTF Web 题目中，密码爆破的目标往往是：

```
登录页面 → 需要密码或验证码
重置密码 → Token 可预测
敏感接口 → 需要特定密钥
```

### 典型场景速查

| 场景 | 常见突破口 | 搜索空间 |
|:----:|-----------|:--------:|
| 验证码生成 | 时间戳种子 | 前后几秒 ~ 几十个种子 |
| 密码重置 Token | 伪随机数 + MD5 | 百万量级 |
| 登录密码 (六位数字) | MD5 爆破 | 一百万 |
| 登录密码 (混合) | 字典攻击 | 取决于字典 |
| 签名校验 | CRC32 或截断哈希 | 极低 |
| API Key | UUID v1 可预测 | 可预测时间戳 |
| Session ID | 短随机值 | 可枚举 |

### 爆破决策树

```
开始爆破
  │
  ├─ 是 PRNG 生成的 Token/验证码？
  │     │
  │     ├─ 确定算法（Python random / PHP mt_rand / Java Random）
  │     ├─ 确定种子范围（时间戳？用户ID？）
  │     ├─ 复现相同调用序列
  │     └─ 枚举种子 → 验证输出
  │
  ├─ 是哈希值比较？
  │     │
  │     ├─ 确定算法 (MD5? SHA1? SHA256?)
  │     ├─ 是否有盐？（位置？分隔符？）
  │     ├─ 哈希完整还是截断？
  │     │     ├─ 完整 → 直接爆破或字典
  │     │     └─ 截断 → 缩小搜索空间，前缀/后缀匹配
  │     └─ 选择工具 (Python / Hashcat)
  │
  └─ 是字典攻击？
        │
        ├─ 已知格式？→ 生成自定义字典
        └─ 未知格式？→ 使用常见字典 (rockyou 等)
```

---

## 二、原理

### 2.1 伪随机数生成器 (PRNG)

普通伪随机数不是真正随机。给定**相同算法 + 相同种子 + 相同调用次数**，输出序列完全一致。

```
种子 (Seed)
  ↓
PRNG 算法 (如 Mersenne Twister)
  ↓
第 1 次调用 → 输出 1
第 2 次调用 → 输出 2
第 3 次调用 → 输出 3
...
```

### 2.2 不同语言的 PRNG 对比

| 语言 | 标准 PRNG | 安全 PRNG | 种子类型 | 状态大小 |
|:----:|:---------:|:---------:|:--------:|:--------:|
| Python | `random.Random()` | `secrets` | 任意 hashable | 19937 bit |
| PHP | `mt_rand()` | `random_int()` | int (32 bit) | 19937 bit |
| PHP | `rand()` | `random_int()` | int (实现相关) | 31 bit |
| Java | `java.util.Random` | `SecureRandom` | long (48 bit) | 48 bit |
| JavaScript | `Math.random()` | `crypto.getRandomValues()` | 实现相关 | 变化 |
| C | `rand()` | - | unsigned int | 32 bit |
| C++ | `std::mt19937` | - | u32 | 19937 bit |
| Go | `math/rand` | `crypto/rand` | int64 | 变化 |

**关键规律**：安全 PRNG（`secrets`、`random_int()`、`SecureRandom`）不能用时间种子预测，普通 PRNG 则可预测。

### 2.3 CTF 中常见弱种子来源

| 种子来源 | 爆破方法 | 搜索范围示例 |
|:--------:|---------|:-----------:|
| 秒级时间戳 `int(time.time())` | 枚举前后秒数 | ±10 秒 = 21 个候选 |
| 毫秒级时间戳 `int(time.time()*1000)` | 枚举前后毫秒 | ±3000 ms = 6001 个候选 |
| 微秒级时间戳 `int(time.time()*1000000)` | 枚举前后微秒 | ±3000000 = 6000001 个候选 |
| 用户 ID | 直接读取或枚举 | 个位数 ~ 千位数 |
| 进程 PID | 枚举常见 PID 范围 | 1~65536 |
| 固定数字 (如 `123456`) | 直接尝试 | 1 |
| 空值或未初始化 | 尝试 0 或默认值 | 几个 |
| 混合种子 `int(time() + pid)` | 组合枚举 | 秒数×PID 范围 |

### 2.4 哈希函数

哈希函数把任意长度输入映射为固定长度摘要：

```
任意输入 → 哈希函数 → 固定长度输出 (如 32 位十六进制)
```

常见哈希长度判断：

| 十六进制长度 | 可能算法 |
|:----------:|---------|
| 32 位 | MD5, NTLM, MD4 |
| 40 位 | SHA-1, RIPEMD-160 |
| 56 位 | SHA-224 |
| 64 位 | SHA-256, BLAKE2s-256 |
| 96 位 | SHA-384 |
| 128 位 | SHA-512, BLAKE2b-512 |

> 注意：仅凭长度不能唯一确定算法，要结合源码判断。

### 2.5 盐 (Salt) 的影响

盐是添加到密码中的随机数据：

```
hash(password + salt)
```

#### 盐的处理方式对比

| 拼接方式 | 示例 | Python 对应 |
|:--------:|:----:|:-----------:|
| 盐在后 | `md5(password + salt)` | `data = pwd + salt` |
| 盐在前 | `md5(salt + password)` | `data = salt + pwd` |
| 带分隔符 | `md5(username + ":" + password)` | `data = username + ":" + pwd` |
| 双重哈希 | `md5(md5(password))` | 先算一次，再算第二次 |
| 固定前缀 | `md5(password + "固定前缀")` | `data = pwd + prefix` |
| HMAC 结构 | `hmac.new(salt, password, hashlib.md5).hexdigest()` | HMAC 标准 |

#### 盐的常见误解

| 误解 | 真相 |
|:----:|:----:|
| 加盐后无法爆破 | 盐不阻止对单个密码的离线爆破 |
| 盐必须保密 | 盐的主要作用是防止预计算和批量复用 |
| 盐越长越安全 | 盐长度影响彩虹表效率，但不影响单次爆破 |
| 固定盐 = 足够好 | 固定盐导致相同密码产生相同哈希 |

盐的作用：
- 防止相同密码产生相同哈希
- 防止预计算彩虹表
-**不阻止对单个弱密码的离线爆破**（因为盐已知时，每个候选密码仍然可以带上盐计算）

### 3.1 伪随机数爆破实战

#### 3.1.1 秒级时间戳种子

**场景**：后端用当前秒级时间戳作为种子生成 6 位验证码。

```python
# 后端代码示意
import random, time
seed = int(time.time())          # 秒级种子
rng = random.Random(seed)
code = f"{rng.randint(0, 999999):06d}"
```

**攻击思路**：已知验证码为 `827103`，生成时间约在 `2026-07-20 12:00:00` 附近。

```python
import random

# 将已知时间转换为时间戳
# date -d "2026-07-20 12:00:00" +%s
center_ts = 1773000000
known_code = "827103"

for offset in range(-30, 31):  # 前后30秒
    seed = center_ts + offset
    rng = random.Random(seed)
    code = f"{rng.randint(0, 999999):06d}"

    if code == known_code:
        print(f"[+] 种子找到: {seed} (偏移: {offset}s)")
        # 使用同一个 rng 对象生成下一次验证码
        next_code = f"{rng.randint(0, 999999):06d}"
        print(f"[+] 下一次验证码: {next_code}")
        break
```

**关键细节**：
- 找到种子后，**必须继续使用同一个 `rng` 对象**生成后续值
- 重新 `Random(seed)` 会从头开始，得到相同序列
- 验证码可能对应多个种子，用两次输出交叉验证

#### 3.1.2 毫秒级时间戳种子

**场景**：后端使用毫秒精度种子。

```python
seed = int(time.time() * 1000)
```

**攻击**：枚举中心时间前后 5000 毫秒（共 10001 个候选）。

```python
import random

center_ms = 1773000000000  # 毫秒级时间戳
known_value = "384921"

for seed in range(center_ms - 5000, center_ms + 5001):
    rng = random.Random(seed)
    val = f"{rng.randint(0, 999999):06d}"

    if val == known_value:
        print(f"[+] 毫秒种子: {seed}")
        break
```

#### 3.1.3 PHP mt_rand 种子爆破

PHP 的 `mt_rand()` 使用 Mersenne Twister，种子为 32 位整数。

```php
<?php
mt_srand(intval(time()));  // 常见弱种子
echo mt_rand(100000, 999999);
?>
```

Python 中复现 PHP 的 mt_rand：

```python
# pip install phpmtseed
import random
import phpmtseed

# 或手工复现 PHP 的 mt_rand 算法
# 注意：Python 的 random 和 PHP 的 mt_rand 实现不同，不能直接混用
```

#### 3.1.4 经过哈希包装的随机数

**场景**：随机数经过 MD5 后再作为 Token。

```python
# 后端代码示意
rng = random.Random(int(time.time()))
value = str(rng.randint(0, 999999))
token = hashlib.md5(value.encode()).hexdigest()
```

虽然最终看到的是 32 位 MD5，但**原始随机数的熵只有 100 万种可能**，加哈希并没有扩大搜索空间。

```python
import hashlib, random

target_token = "a1b2c3d4e5f6...已知的32位MD5".lower()
center_ts = 1773000000

for offset in range(-20, 21):
    seed = center_ts + offset
    rng = random.Random(seed)
    raw = str(rng.randint(0, 999999))
    token = hashlib.md5(raw.encode()).hexdigest()

    if token == target_token:
        print(f"[+] 种子: {seed}, 原始值: {raw}")
        break
```

#### 3.1.5 多步随机调用处理

**场景**：生成 Token 前还调用了其他随机函数。

```php
<?php
mt_srand(intval(time()));
$nonce = mt_rand(0, 9999);       // 第1次调用
$code = mt_rand(100000, 999999); // 第2次调用 → 这是可见的验证码
?>
```

攻击脚本必须模拟第 1 次调用后再取第 2 次：

```python
import random

center_ts = 1773000000

for offset in range(-10, 11):
    seed = center_ts + offset
    rng = random.Random(seed)
    # 模拟第1次调用（即使不可见）
    rng.randint(0, 9999)  # 消耗一次
    # 取第2次调用
    code = rng.randint(100000, 999999)
    # 与目标验证码比较
```

 **新手避坑**：
- `hashlib.md5(value.encode())` 中 `.encode()` 默认 UTF-8，注意编码差异
- Python 3 的 `random` 和 PHP 的 `mt_rand()` 不能混用
- 如果生成 Token 前还有其他随机调用，脚本也要先执行相同次数调用
- 单个输出可能对应多个种子，建议用连续两次输出验证

### 3.2 哈希爆破实战

#### 3.2.1 完整哈希爆破

**场景**：已知 6 位数字密码的 MD5，找回原文。

```python
import hashlib

target = "e10adc3949ba59abbe56e057f20f883e"

for i in range(1_000_000):
    pwd = f"{i:06d}"       # 保留前导零
    digest = hashlib.md5(pwd.encode()).hexdigest()

    if digest == target:
        print(f"[+] 原文: {pwd}")
        break
```

输出：
```
[+] 原文: 123456
```

**前导零陷阱**：`f"{i:06d}"` 保证了 `000000` ~ `999999` 的完整覆盖，直接用 `str(i)` 会漏掉 `000123` 这类密码。

#### 3.2.2 不完整哈希匹配

**场景**：哈希部分未知，如 `e10adc3949ba59abbe56e057f20f88??`。

```python
import hashlib

pattern = "e10adc3949ba59abbe56e057f20f88??".lower()

def match(digest, pattern):
    """逐字符匹配，?表示任意"""
    if len(digest) != len(pattern):
        return False
    return all(p == "?" or d == p for d, p in zip(digest, pattern))

for i in range(1_000_000):
    pwd = f"{i:06d}"
    digest = hashlib.md5(pwd.encode()).hexdigest()

    if match(digest, pattern):
        print(f"[+] 候选: {pwd} -> {digest}")
```

**部分匹配技巧**：
- 前缀已知 → `digest.startswith("e10adc")`
- 后缀已知 → `digest.endswith("f20f883e")`
- 位置已知 → 用上述 `match()` 函数

 **新手避坑**：部分匹配可能返回多个结果，不要找到第一个就退出，收集全部再验证。

#### 3.2.3 带盐哈希爆破

**场景**：后端使用 `md5(password + salt)`。

```python
import hashlib

target = "目标MD5值".lower()
salt = "ctf2026"

for i in range(1_000_000):
    pwd = f"{i:06d}"
    # 注意拼接顺序要和后端一致
    data = pwd + salt
    digest = hashlib.md5(data.encode()).hexdigest()

    if digest == target:
        print(f"[+] 密码: {pwd}")
        break
```

**常见拼接方式对比**：

| 后端写法 | Python 对应 |
|:--------:|:-----------:|
| `md5(password + salt)` | `data = pwd + salt` |
| `md5(salt + password)` | `data = salt + pwd` |
| `md5(username + ":" + password)` | `data = username + ":" + pwd` |
| `md5(md5(password))` | 先算一次，再算第二次 |
| `md5(password + "固定前缀")` | `data = pwd + prefix` |

顺序、分隔符不同，结果完全不同。

#### 3.2.4 使用 Hashcat 加速

当搜索空间很大（如 8 位以上全字符）时，Python 单线程太慢，应使用 GPU 工具。

```bash
# MD5 字典攻击
hashcat -m 0 -a 0 hash.txt rockyou.txt

# MD5 6位数字掩码攻击
hashcat -m 0 -a 3 hash.txt ?d?d?d?d?d?d

# 查看已破解结果
hashcat -m 0 hash.txt --show
```

**Hashcat 常用模式**：

| 算法 | 模式号 | 说明 |
|:----:|:-----:|------|
| MD5 | 0 | 最基础 |
| md5($pass.$salt) | 10 | 盐在后 |
| md5($salt.$pass) | 20 | 盐在前 |
| SHA-1 | 100 | SHA-1 爆破 |
| sha1($pass.$salt) | 110 | SHA-1 盐在后 |
| SHA-256 | 1400 | SHA-256 爆破 |
| SHA-512 | 1700 | SHA-512 爆破 |
| bcrypt ($2*$) | 3200 | 慢哈希 |
| NTLM | 1000 | Windows 哈希 |

#### 3.2.5 Hashcat 实用技巧

```bash
# 组合攻击：字典1 + 字典2
hashcat -m 0 -a 1 hash.txt dict1.txt dict2.txt

# 混合攻击：字典 + 掩码后缀
hashcat -m 0 -a 6 hash.txt dict.txt ?d?d

# 字典 + 规则（常见变形）
hashcat -m 0 -a 0 hash.txt rockyou.txt -r rules/best64.rule

# 指定自定义字符集
hashcat -m 0 -a 3 hash.txt -1 ?l?u ?1?1?1?1?1?1?1?1

# 启用 potfile 自动保存进度
hashcat -m 0 hash.txt rockyou.txt --potfile-path=my.pot

# 仅显示已破解结果（不重新跑）
hashcat -m 0 hash.txt --show
```

### 3.3 字典生成与使用实战

#### 3.3.1 根据格式生成字典

**场景**：密码格式为 `admin` + 4 位数字，共 10000 种可能。

```python
# 生成 admin0000 ~ admin9999
with open("dict.txt", "w", encoding="utf-8") as f:
    for i in range(10000):
        f.write(f"admin{i:04d}\n")
```

**场景**：密码每位使用不同字符集。

```python
import itertools, string

char_sets = [
    string.ascii_uppercase,  # 第1位: A-Z
    string.digits,           # 第2位: 0-9
    "abc",                   # 第3位: a,b,c
    "@_",                    # 第4位: @或_
]

# 先估算数量，避免生成超大文件
total = 1
for cs in char_sets:
    total *= len(cs)
print(f"预计生成 {total} 个候选")
# 26 * 10 * 3 * 2 = 1560

with open("dict.txt", "w", encoding="utf-8") as f:
    for combo in itertools.product(*char_sets):
        f.write("".join(combo) + "\n")
```

#### 3.3.2 使用 Crunch 工具

```bash
# 6位纯数字
crunch 6 6 0123456789 -o 6digit.txt

# 4~6位小写字母+数字
crunch 4 6 abcdefghijklmnopqrstuvwxyz0123456789 -o wordlist.txt

# 指定输出格式
crunch 6 6 -t admin@@ -o admin_dict.txt  # @代表数字
```

#### 3.3.3 字典读取要点

```python
import hashlib

target = "目标哈希".lower()

with open("dict.txt", "r", encoding="utf-8", errors="ignore") as f:
    for line in f:
        pwd = line.rstrip("\r\n")    # 只移除换行
        #  不要用 line.strip()，会移除密码中原有的空格
        digest = hashlib.md5(pwd.encode()).hexdigest()

        if digest == target:
            print(f"[+] 找到: {pwd}")
            break
```

 **新手避坑**：
- `line.strip()` 会删除密码首尾的空格，可能造成漏报
- `line.rstrip("\r\n")` 只删除换行符
- 注意文件编码，Windows 下可能是 `gbk` 或 `utf-8`

#### 3.3.4 在线 vs 离线爆破选择

| 方式 | 工具 | 速度 | 受限制 | 适用场景 |
|:----:|:----:|:----:|:------:|:--------:|
| 离线 | Python / Hashcat | 快 | 无 | 已知哈希值 |
| 在线 | BurpSuite Intruder | 慢 | 限速/验证码 | 需要提交给 Web 接口 |
| 在线 | hydra | 中等 | 限速/锁定 | 表单/服务爆破 |
| 在线 | medusa | 中等 | 限速/锁定 | 各类协议爆破 |

**原则**：能离线就离线。BurpSuite 在线爆破适合小候选集（几百个）。

## 四、避坑汇总

| 编号 | 坑 | 正确做法 |
|:---:|----|---------|
| 1 | 忘记保留前导零 | 用 `f"{i:06d}"` 确保固定宽度 |
| 2 | 行读取用 `strip()` | 改用 `rstrip("\r\n")` |
| 3 | 找到第一个匹配就退出 | 部分匹配时收集所有候选 |
| 4 | 不同语言 PRNG 混用 | 确认算法和平台一致 |
| 5 | 忽略调用顺序 | 脚本中必须执行未观察到的随机调用 |
| 6 | 忘记区分秒/毫秒 | 确认种子是 `int(time.time())` 还是 `int(time.time()*1000)` |
| 7 | `itertools.product` 转列表 | 保持迭代器，不要 `list(product(...))` |
| 8 | 哈希大小写不统一 | 全部 `.lower()` 后再比较 |
| 9 | 在线爆破盲目发送大量请求 | 先离线缩小候选范围 |
| 10 | Python 跑大搜索空间 | 换 Hashcat 用 GPU |
| 11 | 忽视盐的拼接顺序 | 先通过源码确认盐在前还是后 |
| 12 | 混用安全/非安全 PRNG | `secrets` 和 `random` 不能互换 |
| 13 | 以为 MD5(随机数) 安全 | 输入熵不变，哈希不增加搜索空间 |
| 14 | 一次验证码匹配就确认种子 | 用两次连续输出交叉验证 |
| 15 | 大字典直接全量在线爆破 | 先生成小字典离线过滤再上线 |

## 五、知识总结表

### 各攻击类型速查

| 攻击类型 | 适用条件 | 核心工具 | 代码关键点 |
|:--------:|---------|:--------:|:----------:|
| 时间种子爆破 | Token/验证码由 PRNG + 时间戳生成 | Python `random` | 复现相同 `Random(seed)` + 相同调用序列 |
| 完整哈希爆破 | 已知哈希值和算法 | Python `hashlib` / Hashcat | 注意编码、盐位置、前导零 |
| 不完整哈希匹配 | 哈希部分缺失 | Python 逐字符匹配 | 收集全部候选 |
| 带盐哈希爆破 | 已知盐值和拼接方式 | Python / Hashcat | 拼接顺序必须与后端一致 |
| 字典攻击 | 未知密码但可能有弱口令 | Hashcat / John | 可先生成自定义字典 |
| 掩码攻击 | 已知密码结构 | Hashcat `?d?d?d` | 搜索空间 = 各位置字符数乘积 |

### 常用 Hashcat 参数

| 参数 | 含义 | 示例 |
|:----:|:----:|:----:|
| `-m` | 哈希模式 | `-m 0` (MD5) |
| `-a` | 攻击模式 | `-a 0` (字典), `-a 3` (掩码) |

| `--force` | 忽略警告 | 部分环境需要 |

### Hashcat 掩码字符集

| 占位符 | 含义 | 等价的字符 |
|:-----:|:----:|:---------:|
| `?d` | 数字 | `0123456789` |
| `?l` | 小写字母 | `abcdefghijklmnopqrstuvwxyz` |
| `?u` | 大写字母 | `ABCDEFGHIJKLMNOPQRSTUVWXYZ` |
| `?s` | 特殊符号 | `!"#$%&'()*+,-./:;<=>?@[\]^_{|}~` |
| `?a` | 所有可打印字符 | `?l?u?d?s` |
| `?b` | 所有字节 0x00-0xFF | 暴力穷举所有字节 |

### 快速估算参考

| 密码长度 | 字符集 | 搜索空间 | Python 耗时估算 | Hashcat 耗时估算 |
|:--------:|:-----:|:--------:|:--------------:|:---------------:|
| 6 位 | 纯数字 | 10^6 | 几秒 | 毫秒 |
| 8 位 | 小写字母 | 26^8 ≈ 2×10^11 | 几周 | 几小时 |
| 8 位 | 全字符 | 95^8 ≈ 6.6×10^15 | 不可行 | 可能需要 GPU 集群 |
| 4 位 | 小写+数字 | 36^4 ≈ 1.7×10^6 | 几秒 | 毫秒 |
| 10 位 | 纯数字 | 10^10 | 不可行 | 几分钟 |
| 8 位 | 数字+字母 | 62^8 ≈ 2×10^14 | 不可行 | 可能需要多 GPU |

> 估算依据：Python 约 50 万 MD5/s，Hashcat 约 100 亿 MD5/s (RTX 4090)
